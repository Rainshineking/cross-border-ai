## 1. 项目概述

### 1.1 项目背景

本项目为参加 **中国国际大学生创新大赛** 而设计。核心切入点是：利用自主研发的 AI 工具，实现跨境电商全流程的 **自动化选品、自动化投流、自动化创意生成**。用户只需输入一句话需求，系统即可自主完成信息检索、分析、推荐与内容生成。

### 1.2 核心价值

| 维度 | 说明 |
|------|------|
| **自动化** | 从"人工调研-人工分析-人工制作"转变为"AI 理解需求→AI 检索→AI 分析→AI 推荐"的闭环 |
| **智能化** | 基于大语言模型 + 联网搜索，实时感知市场趋势 |
| **低门槛** | 用户无需数据分析技能，用自然语言即可获得专业选品建议 |
| **全流程** | 覆盖选品、投流、创意三大跨境电商核心环节 |

### 1.3 目标用户

- 跨境电商卖家（亚马逊、Temu、1688 等平台）
- 跨境创业新手
- 选品运营人员

## 2. 竞品分析 & 创新点

### 2.1 现有方案痛点

| 现有工具 | 局限性 |
|----------|--------|
| Jungle Scout / Helium 10 | 仅提供数据面板，需人工分析决策，无 AI 对话交互 |
| ChatGPT 通用问答 | 不具备跨境电商领域知识，无法联网实时检索商品数据 |
| 传统数据大屏 | 只展示数据，不提供 actionable 的推荐建议 |

### 2.2 我们的创新点

1. **AI Agent 驱动**：用户用自然语言与 AI 对话，AI 自主联网搜索、分析并推荐产品
2. **1+1 双形态**：AI 对话助手（便捷入口）+ 数据驾驶舱（深度分析），兼顾效率与深度
3. **全流程覆盖**：选品 → 投流 → 创意生成，一个 AI 助手串联全部环节
4. **轻量级部署**：无需数据库，会话级上下文记忆，Docker 一键部署

## 3. 技术栈选型

### 3.1 总体选型

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **前端框架** | Vue 3 + TypeScript | 组合式 API 开发效率高，生态成熟 |
| **UI 组件库** | Element Plus | 企业级组件丰富，适合数据展示类应用 |
| **状态管理** | Pinia | Vue 3 官方推荐，轻量且类型安全 |
| **HTTP 客户端** | Axios | 广泛使用，拦截器机制便于统一处理 |
| **图表库** | ECharts | 数据大屏可视化能力强 |
| **后端框架** | Spring Boot 3 | Java 生态成熟，社区活跃 |
| **AI Agent 框架** | LangChain4j 或 Alibaba Cloud AI | 两者均支持 Java 端的 LLM 编排与工具调用 |
| **向量存储（可选）** | Chroma (嵌入式) | 如需 RAG 增强可引入，无需额外部署 |
| **构建工具** | Maven | 项目标准构建工具 |
| **容器化** | Docker + Docker Compose | 统一部署环境，便于云服务器运行 |
| **CI/CD** | GitHub Actions（可选） | 自动化构建与部署 |

### 3.2 AI Agent 框架对比

| 对比维度 | LangChain4j | Alibaba Cloud AI (百炼) |
|----------|-------------|----------------------|
| **开源** | ✅ 完全开源 | ❌ 部分开源 |
| **LLM 支持** | 多模型（OpenAI、文心、通义等） | 通义系列为主 |
| **工具/函数调用** | ✅ 原生支持 | ✅ 支持 |
| **RAG** | ✅ 内置支持 | ✅ 支持 |
| **社区活跃度** | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **国内网络友好** | ⭐⭐⭐（需配代理） | ⭐⭐⭐⭐⭐ |
| **Spring Boot 集成** | ⭐⭐⭐⭐⭐ 原生支持 | ⭐⭐⭐⭐ |

**推荐方案**：主选 **LangChain4j** + 通义千问 API（兼容 OpenAI 协议），既可复用社区生态，又能在国内无障碍调用。

### 5.2 AI 对话模块

**核心功能**：
- 接收用户自然语言输入
- 利用 LLM 理解用户意图（选品查询 / 投流咨询 / 创意需求）
- 根据意图自动调用对应工具（联网搜索、数据分析等）
- 整合工具返回结果，生成结构化回答
- 维护会话级上下文，支持多轮追问

**关键技术**：
- **Function Calling**：定义领域工具函数（analyze_trend, generate_ad_copy 等），选品搜索直接由模型内置的联网搜索能力完成
- **System Prompt 工程**：固定角色设定 + 动态注入上下文
- **流式输出**：Server-Sent Events (SSE) 实现流式响应，提升用户体验

### 5.3 选品推荐模块

**选品评分卡指标（12 项）**：

| 指标 | 权重 | 说明 | 数据来源 |
|------|------|------|---------|
| 市场热度 | 15% | 近 7 天搜索增长趋势 | 联网搜索 |
| 竞争强度 | 15% | 在售商品数量 / 头部品牌集中度 | 联网搜索 |
| 毛利率潜力 | 10% | 售价区间与采购成本对比 | 联网搜索 |
| 供应链匹配度 | 10% | 国内供应商库存与起订量 | 联网搜索 |
| 评论数趋势 | 8% | 近期评论增长量 | 联网搜索 |
| 评分表现 | 8% | 平均评分 4.0 以上占比 | 联网搜索 |
| 价格带分布 | 8% | 主流价格区间与利润空间 | 联网搜索 |
| 季节性波动 | 6% | 全年销量波动幅度 | 联网搜索 |
| 品牌垄断度 | 6% | TOP3 品牌市场份额 | 联网搜索 |
| 物流适配性 | 5% | 产品尺寸重量与 FBA 费用 | 联网搜索 |
| 法律法规风险 | 5% | 品类合规要求与认证门槛 | 联网搜索 |
| 复购率 | 4% | 自然复购 / 消耗品属性 | 联网搜索 |

### 5.4 投流建议模块

- **渠道推荐**：亚马逊 SP/SB/SD 广告、TikTok Shop、Facebook Ads 等
- **策略生成**：基于产品类目与目标市场，自动生成广告投放策略
- **关键词推荐**：自动提取高转化关键词与长尾词
- **预算分配**：建议的测试期预算与放量节奏

### 5.5 创意生成模块

- **广告文案**：基于产品卖点生成多版本 A+ 文案
- **卖点提炼**：从产品描述与评论中提取核心卖点
- **关键词优化**：推荐 SEO 关键词与后台关键词
- **图像描述**：生成广告图的 Prompt 描述（可对接文生图 API）

## 6. AI 智能体设计

### 6.1 Agent 角色设定（System Prompt）

```
你是一位专业的跨境电商选品与投流专家。你的名字是"跨境选品小助手"。

你的核心能力：
1. 选品分析：根据用户需求，通过联网搜索获取最新的市场数据，分析品类热度、竞争情况、利润空间等
2. 投流建议：基于产品特点，提供广告投放策略和渠道建议
3. 创意生成：为产品生成广告文案、卖点提炼和关键词推荐

行为准则：
- 始终以跨境电商选品专家的身份回答
- 当用户提出选品需求时，优先联网搜索获取最新数据
- 给出推荐时，需要附上数据支撑和理由
- 回答结构清晰，使用表格、列表等格式
- 如果用户问题超出跨境电商范围，礼貌引导回主题

当前会话上下文：
{{会话历史摘要}}
```

### 6.2 工具定义（Function Calling）

| 工具名称 | 功能 | 参数 |
|---------|------|------|
| `analyze_trend` | 分析品类趋势 | `category`、`time_range` |
| `get_product_details` | 获取产品详情 | `product_id`、`platform` |
| `generate_ad_copy` | 生成广告文案 | `product_name`、`features`、`target_audience` |
| `recommend_keywords` | 推荐关键词 | `product_name`、`category`、`platform` |

> 注：联网搜索功能不再单独定义工具，而是通过通义千问模型自带的 `enable_search` 参数开启，模型会自动判断何时需要联网检索。

### 6.3 对话流程设计

```
User Input
    │
    ▼
┌──────────────────────┐
│  LLM 处理             │
│  (通义千问 + 联网搜索)  │
│  自动识别意图          │
│  按需联网检索          │
│  生成结构化回答        │
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│ 结构化输出             │
│ (JSON → 前端渲染)      │
└──────────────────────┘
```

### 6.4 上下文管理

- **方案**：内存存储（无需数据库）
- **存储内容**：
  - 本次会话的消息历史（List<Message>）
  - 会话摘要（每 N 轮对话自动压缩摘要）
  - 当前选中品类/产品（如有）
- **会话标识**：前端生成 UUID，每次请求携带
- **生命周期**：后端服务重启后丢失（可通过 Redis 持久化，但非 MVP 必需）

### 9.2 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # SSE 支持（关键）
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
    }
}
```
